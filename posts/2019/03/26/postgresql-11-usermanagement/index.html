PostgreSQLのユーザ管理

sudo -u postgres createuser --no-createdb --login --pwprompt --no-createrole --no-superuser --pwprompt testuser

psqlで接続できるか確認する。

psql -U testuser testdb

DBに関しては特に指定は無くても、存在すればアクセスできる。テーブルも作れる。

このようにDB作成直後はユーザが割と何でもできるので、一旦権限を取り上げて特定のDBに対してのみ権限を与える。

sudo -u postgres psql  << '__EOF__'
REVOKE CONNECT ON DATABASE testdb FROM PUBLIC; -- 指定したDBから接続権限を取り上げる
REVOKE ALL ON SCHEMA public FROM PUBLIC; -- 全てのpublicスキーマからすべての権限を取り上げる
__EOF__

上記のpsqlコマンドで接続できなくなったことを確認する。

sudo -u postgres psql  << '__EOF__'
GRANT CONNECT ON DATABASE testdb TO testuser; -- 指定したDBへ指定したユーザの接続を許可
GRANT USAGE ON SCHEMA public TO testuser;-- 指定したユーザへpublicスキーマの操作の許可
__EOF__

該当DBのpublicスキーマを使ったテーブルの作成・削除などはできるようになる。
